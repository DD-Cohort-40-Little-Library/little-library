Content Strategy

This website will provide the local Albuquerque community with a tool to better connect neighborhoods to the Little Libraries and those who use them and maintain them. Additionally, this application will encourage literacy and literary event planning, promotion and attendance. In addition, this site will:

  • Let users know, once they are logged in, what Little Libraries and public libraries are within walking distance
  • The Who, What, When and Where of literacy and literary events
  • The ability to search for specific types of Little Libraries
  • The ability, as a Steward of a Little Library, to manage their Little Library

The map is critical to the project and will be the central feature of the site.

Landing Page

Map, event sidebar, nav sidebar and map filters

  • Desktop view
    • The map is centered on the page and illustrates Little Libraries by map pins
      • By clicking on a map pin the user will be presented with a card for the Little Library location, providing:
        • Photo of the Little Library (if available)
        • Name
        • Address
        • Specialization (if any)
        • buttons for:
          • check-in
          • Report issue
          • Upload image
    • The left sidebar contains a chronological listing of events, presenting the events first by date and then by distance from the user.
      • The events are clickable and when clicked will expand to provide the full information of the event and enlarges a map pin for the location of the event
      • At the top of the list of events is a button to "Plan An Event"
        • Clicking this will open a modal with a form for listing an event
    • The right sidebar contains the navigation buttons for User Login, LL Steward Page and Registration
      • Clicking the user login button opens a modal allowing the user to log into the site.
      • Clicking the Steward button will redirect the user to the Steward page.
      • Clicking the registration button will open a modal allowing the user to register for an account on the site
    • Below the buttons are a search bar for zip, state, city filtering and the radial buttons for filter options that the user can select for the map which are triggered from the search button
    • The header contains the logo to the top left with the name of the site centered over the map
    • The footer will contain credits for the project and links to any additional information relevant to site and the team.
  • Mobile view
    • The map is centered at the page and illustrates Little Libraries by map pins
      • By clicking on a map pin the user will be presented with a card for the Little Library location, providing:
        • Photo of the Little Library (if available)
        • Name
        • Address
        • Specialization (if any)
        • buttons for:
          • check-in
          • Report issue
          • Upload image
    • Below the map are the navigation buttons for user login, steward page and registration
    • Below the buttons are a search bar for zip, state, city filtering and the radial buttons for filter options that the user can select for the map which are triggered from the search button
    • Next is a chronological listing of events, presenting the events first by date and then by distance from the user.
      • The events are clickable and when clicked will expand to provide the full information of the event and enlarges a map pin for the location of the event on the map
      • At the top of the list of events is a button to "Plan An Event"
        • Clicking this will open a modal with a form for listing an event
    • The header contains the logo to the top left with the name of the site centered over the map and the navigation button for login, steward and register will be collapsed in the right edge of the header
      • Clicking the user login button opens a modal allowing the user to log into the site.
      • Clicking the Steward button will redirect the user to the Steward page.
      • Clicking the registration button will open a modal allowing the user to register for an account on the site
    • The footer will contain credits for the project and links to any additional information relevant to site and the team.

Steward Page

    The steward page will provide library stewards with the ability to manage their library in a variety of ways

  • Desktop view
    • Stewards will see the following buttons that will open a modal to perform the following actions:
      • A list of the libraries they are associated with and can be clicked on to:
        • Add
        • Move
        • Close
        • Edit
    • Possibly use a modal that exposes the full list when hovered over
  • Mobile view
    • The lists will collapse to a drop-down clickable list that will either expand or open a modal view and will be labeled with Soft and Tech respectively.

Portfolio

The portfolio section will provide six examples of work. The work will be presented in clickable thumbnail images linking to the project depicted in the thumbnail. Each thumbnail will have a semi-transparent label with the title of the example of work. When clicked the location will open in a new tab so that the user can easily move between the main page and the location of the work.

Below the thumbnail will be a link to the GitHub repository for the code of the featured project. These links will also open in a new tab.

  • Desktop view
    • The thumbnails are arranged in two rows of three evenly spaced with the middle thumbnail centered on the page.
    • Immediately below the thumbnail will be a link to the GitHub repository centered on the above thumbnail.
  • Mobile view
    • The thumbnails will move to display as three rows of two, evenly spaced with the GitHub link centered on the thumbnail image just under each thumbnail.

Additional information section

The information provided here will only be relevant if the Welcome section hooks the viewer and should be below the scroll.

Education

My education will be split between those that are relevant to the technical position I am seeking, this section, and the Other section where non-technical, less relevant education will be listed.

  • Desktop view
    • Will provide relevant education starting the with the most recent, Deep Dive Coding, and listing the two Bachelors that with the school attended and dates of attendance.
    • Each entry will occupy a single line with type of education, institution and dates of attendance.
  • Mobile view
    • The mobile view will present the same information but will split the information into additional lines to improve line length and readability.

Certification

This section will list all relevant IT certifications that I have received.

  • Desktop view
    • The certifications will be listed in two columns evenly spaced and parallel
  • Mobile view
    • The list will stack on the mobile view in a single column aligned to the left

Experience

This section will show my last 12 years of work experience, demonstrating longevity and strong, high level responsibility and work.

  • Desktop view
    • This will be a list of the positions held over the last 12 years with business, location, title and dates of employment
    • The list will be left aligned with a large white space on either side. The first line of each block has the business name and location with the title and dates indented on a new line. This provides some asymmetry to help the eye flow
  • Mobile view
    • Will appear similar to the desktop view but the white space reduced to fit the screen but continuing the same left aligned first line and the additional lines indented.

Other

This section contains the unrelated education in the same format as the relevant education and also includes my Martial Arts ranking as a demonstration of interests and accomplishments.

  • Desktop view
    • The unrelated education will mirror the format of the previous education section
    • The martial arts section will mirror the work experience layout used previously with white space and left aligned first line and indented sub items.
  • Mobile view
    • The unrelated education in mobile view will mirror the mobile view of the previous education section, splitting the information into lines per information type instead of on a single line.
    • Similar to the previous work experience section, the martial arts section will maintain the pattern but without the use of the white space on either side.

Footer

The footer will be a thick black bar along the bottom and will contain linking icons for LinkedIn, the envelope icon in the center for activating the email contact modal, and FaceBook.

This is consistent in both the desktop and mobile views.
