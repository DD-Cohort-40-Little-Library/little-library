Data Conceptual Model

This is the data conceptual model for the Little Library web application that will provide information and access to Little Free Libraries and tie them together with public libraries in the central Albuquerque area.

Relationships

user

  • The user entity has the following relationships to the other entities and is the strongest entity and is constructed first in the database:
    • user has a one-to-many relationship with the library entity
    • user has a one-to-many relationship with the events entity
    • user has a one-to-many relationship with the checkIn entity

library

  • The library entity has the following relationships to the other entities and is the second strongest entity and is constructed second in the database:
    • library has a many-to-one relationship to the user entity
    • library has a one-to-many relationship to the photos events entity
    • library has a one-to-many relationship to the events entity

events

  • The events entity has the following relationships to the other entities and is the third strongest entity and constructed third in the database:
    • events has a many-to-one relationship with user
    • events has a many-to-one relationship with library

checkIn

  • The checkIn entity has the following relationships to the other entities and is the weakest entity and is the fourth and final entity to be constructed in the database:
    • checkIn has a many-to-one relationship with user
    • checkIn has a many-to-one relationship with library
